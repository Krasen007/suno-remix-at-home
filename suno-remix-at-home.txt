**Suno API Remix Script**

Simple Single-User Implementation Plan

_GitHub Raw → Suno API → Remix MP3_

**Overview**

This is a minimal, single-user script. No server, no database, no job queue. You host your MP3s on a public GitHub repository, run a Python or Node script locally that calls the Suno API, and poll for results until the remix is done. That's it.

**✅ What this replaces from the original plan**

No S3 / cloud storage (GitHub Raw is the file host). No database (a simple JSON file or just console output). No webhook server (polling instead). No job queue. No backend framework.

**Step 1 - Host Your MP3s on GitHub**

GitHub Raw serves files directly from a public repository as a plain HTTPS URL, which is exactly what Suno's uploadUrl parameter requires.

## **How to get a GitHub Raw URL**

- Create a public GitHub repository (e.g. my-remixes)
- Add your MP3 file to the repo - push it to main branch
- Open the file on GitHub, click the Raw button
- Copy the URL - it will look like this:

<https://raw.githubusercontent.com/YOUR_USERNAME/YOUR_REPO/main/my-song.mp3>

**⚠️ Important - GitHub Raw requirements**

The repository MUST be public. Private repos return 404 to external services like Suno. File size limit for GitHub is 100MB - well within any 3-minute MP3 (typically 5-10MB). GitHub Raw URLs are stable and permanent as long as the file exists in the repo.

## **Committing large-ish audio files**

For MP3 files under 50MB, a regular git push works fine. If you ever go over 50MB, use Git LFS - but for 3-minute MP3s at standard bitrate you won't need it.

git clone <https://github.com/YOUR_USERNAME/my-remixes>

cd my-remixes

cp ~/Desktop/my-song.mp3 .

git add my-song.mp3

git commit -m "add source track"

git push

\# Your URL is now:

\# <https://raw.githubusercontent.com/YOUR_USERNAME/my-remixes/main/my-song.mp3>

**Step 2 - The Script**

A single Python script handles everything: checks credits, submits the remix job, polls for completion, and prints the result URL. No dependencies beyond the requests library.

## **Full Python Script (remix.py)**

import requests

import time

import json

\# ─── CONFIG ──────────────────────────────────────────────

SUNO_API_KEY = "YOUR_API_KEY_HERE"

BASE_URL = "<https://api.sunoapi.org/api/v1>"

TRACKS = \[

{

"uploadUrl": "<https://raw.githubusercontent.com/YOU/REPO/main/song1.mp3>",

"title": "Song 1 (Remix)",

"style": "Electronic, Synth Pop",

"prompt": "Transform into an upbeat electronic remix",

},

{

"uploadUrl": "<https://raw.githubusercontent.com/YOU/REPO/main/song2.mp3>",

"title": "Song 2 (Jazz Remix)",

"style": "Smooth Jazz",

"prompt": "Reimagine as a smooth late-night jazz track",

},

\]

HEADERS = {

"Authorization": f"Bearer {SUNO_API_KEY}",

"Content-Type": "application/json",

}

\# ─── HELPERS ─────────────────────────────────────────────

def check_credits():

r = requests.get(f"{BASE_URL}/get-credits", headers=HEADERS)

credits = r.json()\["data"\]\["credits"\]

print(f"Remaining credits: {credits}")

return credits

def submit_remix(track):

payload = {

"uploadUrl": track\["uploadUrl"\],

"customMode": True,

"instrumental":False,

"model": "V5",

"style": track\["style"\],

"title": track\["title"\],

"prompt": track\["prompt"\],

}

r = requests.post(f"{BASE_URL}/generate/upload-cover",

headers=HEADERS, json=payload)

data = r.json()

if data\["code"\] != 200:

raise Exception(f"Submit failed: {data\['msg'\]}")

task_id = data\["data"\]\["taskId"\]

print(f" Submitted → taskId: {task_id}")

return task_id

def poll_until_done(task_id, timeout_sec=600):

print(f" Polling for taskId: {task_id}")

deadline = time.time() + timeout_sec

while time.time() < deadline:

r = requests.get(

f"{BASE_URL}/generate/record-info",

params={"taskId": task_id},

headers=HEADERS

)

data = r.json()\["data"\]

status = data.get("status", "UNKNOWN")

print(f" status: {status}")

if status == "SUCCESS":

tracks = data\["response"\]\["data"\]

return tracks

elif status == "FAILED":

raise Exception(f"Task failed: {data.get('errorMessage', 'unknown')}")

time.sleep(30) # poll every 30 seconds

raise Exception("Timed out after 10 minutes")

\# ─── MAIN ────────────────────────────────────────────────

def main():

print("=== Suno Remix Script ===\\n")

check_credits()

results = {}

for i, track in enumerate(TRACKS):

print(f"\\n\[{i+1}/{len(TRACKS)}\] Processing: {track\['title'\]}")

try:

task_id = submit_remix(track)

variants = poll_until_done(task_id)

results\[track\["title"\]\] = variants

print(f" ✅ Done! Got {len(variants)} variant(s):")

for v in variants:

print(f" {v\['title'\]} - {v\['duration'\]}s")

print(f" URL: {v\['audio_url'\]}")

except Exception as e:

print(f" ❌ Error: {e}")

results\[track\["title"\]\] = {"error": str(e)}

\# Save results to file

with open("remix_results.json", "w") as f:

json.dump(results, f, indent=2)

print("\\n=== All done. Results saved to remix_results.json ===")

if \__name__ == "\__main_\_":

main()

**Step 3 - API Call Parameters**

Here is exactly what gets sent to Suno for each track, and why each parameter is set the way it is:

| **Parameter** | **Value** | **Notes** |
| --- | --- | --- |
| **uploadUrl** | GitHub Raw URL | The public HTTPS URL of your MP3. Suno downloads the file from here. |
| **customMode** | true | Gives full control over style and title. Use false for a simpler auto-everything mode. |
| **instrumental** | false | Keeps vocals in the remix. Set to true if you want an instrumental-only output. |
| **model** | V5  | Latest model. Supports up to 8 minutes. Works fine with 3-minute tracks. |
| **style** | your style | Genre/vibe description. E.g. 'Smooth Jazz', 'Electronic, Synth Pop'. Max 1000 chars. |
| **title** | your title | Name of the remix. Max 100 chars for V5. |
| **prompt** | your prompt | In customMode=true + instrumental=false, this is the EXACT LYRICS that will be sung. If you don't want to control lyrics, set customMode to false and use prompt as a style description instead (max 500 chars). |

**💡 Simple mode vs Custom mode - which to use?**

Custom mode (shown above): you control style, title, and lyrics exactly. Good if you want a specific genre transformation. Simple mode (customMode: false): just provide a prompt like 'make this a jazz remix' and Suno handles everything automatically. For just remixing a few tracks, simple mode is even easier.

## **Simple Mode alternative (even less config)**

payload = {

"uploadUrl": track\["uploadUrl"\],

"customMode": False, # Simple mode

"instrumental": False,

"model": "V5",

"prompt": "Transform into a smooth jazz remix", # max 500 chars

}

**Polling vs Callback - Why Polling Is Fine Here**

The original plan used webhooks (callbacks) because a multi-user server needs to handle many concurrent jobs without blocking. For a single-user script, polling is simpler and perfectly adequate.

|     | **Webhooks (original plan)** | **Polling (this plan) ✓** |
| --- | --- | --- |
| **Setup** | Requires public HTTPS server, SSL, endpoint | None - runs locally |
| **Works offline?** | No - Suno must reach your server | Yes - you reach Suno |
| **Complexity** | Express/Flask server + routes | A while loop |
| **Good for** | Production, many users | **Personal scripts ✓** |
| **API credits wasted?** | No  | No - polling the status endpoint is free |

The polling interval in the script is 30 seconds - the minimum recommended by Suno to avoid rate limiting (error 430). A 3-minute track remix typically takes 2-5 minutes, so expect 4-10 poll cycles.

**What You Get Back**

Suno returns TWO remix variants per submission. The script saves everything to remix_results.json. Example output:

{

"Song 1 (Remix)": \[

{

"id": "abc123",

"title": "Song 1 (Remix)",

"audio_url": "<https://cdn.sunoapi.org/audio/abc123.mp3>",

"duration": 187.4,

"tags": "electronic, synth pop"

},

{

"id": "def456",

"title": "Song 1 (Remix)",

"audio_url": "<https://cdn.sunoapi.org/audio/def456.mp3>",

"duration": 183.1,

"tags": "electronic, synth pop"

}

\]

}

**⚠️ Download your remixes within 15 days**

Suno deletes generated audio files after 15 days. Download the MP3s from the audio_url links and save them locally. The script does not auto-download - just add a requests.get(audio_url) and write to a file if you want that.

## **Optional: Auto-download remixes**

Add this to the script after poll_until_done returns to automatically save the files:

import os

def download_remix(audio_url, filename):

r = requests.get(audio_url)

os.makedirs("remixes", exist_ok=True)

path = f"remixes/{filename}.mp3"

with open(path, "wb") as f:

f.write(r.content)

print(f" Saved: {path}")

\# After poll_until_done:

for i, v in enumerate(variants):

download_remix(v\["audio_url"\], f"{track\['title'\]}\_v{i+1}")

**Error Codes to Know**

| **Code** | **Meaning** | **Fix** |
| --- | --- | --- |
| **400** | Bad request | Check parameters. If customMode=true + instrumental=false, you must provide style, title, and prompt. |
| **401** | Invalid API key | Check your SUNO_API_KEY value. |
| **413** | Prompt too long | Trim your prompt/style/title to within character limits. |
| **429** | Insufficient credits | Buy more credits on sunoapi.org dashboard. |
| **430** | Polling too fast | Increase polling interval to 30+ seconds. |
| **451** | Can't download audio | Make sure the GitHub repo is PUBLIC and the file URL is correct. Test the URL in a browser. |
| **500** | Server error | Wait 30 seconds, retry once. |

**Quick Start Checklist**

- Create a public GitHub repo and push your MP3 files
- Grab each file's Raw URL from GitHub (click Raw button)
- Get your Suno API key from sunoapi.org/api-key
- Copy the remix.py script and fill in SUNO_API_KEY and TRACKS
- Run: pip install requests && python remix.py
- Wait 2-5 minutes per track - watch the polling output
- Open remix_results.json - copy the audio_url links to download your remixes
- Download remixes within 15 days before Suno deletes them
